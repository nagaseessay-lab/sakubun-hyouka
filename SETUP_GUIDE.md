# 作文評価システム セットアップ完全ガイド

**対象**: 初めてセットアップする方（技術知識不要）
**所要時間**: 約1〜2時間
**必要なもの**: Googleアカウント（Google Workspace推奨）、PCとブラウザ

---

## 全体の流れ

```
ステップ1: Googleの設定（ログイン機能に必要）
ステップ2: スプレッドシート作成（データベース）
ステップ3: Google Driveフォルダ作成（PDF保存場所）
ステップ4: Apps Scriptにコードを貼り付け（サーバー）
ステップ5: Apps Scriptをデプロイ（公開）
ステップ6: フロントエンドを公開（Netlify）
ステップ7: 初期ユーザー登録
ステップ8: 動作確認
```

---

## ステップ1: Google Cloud Console の設定

これは「Googleログイン」機能を使うために必要な設定です。

### 1-1. Google Cloud Console を開く

1. ブラウザで https://console.cloud.google.com/ を開きます
2. Googleアカウントでログインします
3. 初めての場合は利用規約に同意します

### 1-2. プロジェクトを作成する

1. 画面上部の「プロジェクトを選択」をクリック
2. 右上の「新しいプロジェクト」をクリック
3. プロジェクト名に「作文評価システム」と入力
4. 「作成」をクリック
5. 作成されたプロジェクトが選択されていることを確認

### 1-3. OAuth同意画面を設定する

1. 左メニューから「APIとサービス」→「OAuth同意画面」をクリック
2. **Google Workspace を使っている場合**: 「内部」を選択 → 「作成」
   **個人のGmailの場合**: 「外部」を選択 → 「作成」
3. 以下を入力:
   - アプリ名: `作文評価システム`
   - ユーザーサポートメール: 自分のメールアドレスを選択
   - デベロッパーの連絡先メールアドレス: 自分のメールアドレスを入力
4. 「保存して次へ」をクリック
5. スコープの画面: そのまま「保存して次へ」
6. テストユーザーの画面:
   - **「外部」を選んだ場合**: 「+ ADD USERS」をクリックして、テストに使うGoogleメールアドレスを全て追加してください（本番公開するまではテストユーザーだけがログインできます）
   - **「内部」を選んだ場合**: そのまま「保存して次へ」
7. 概要の画面: 「ダッシュボードに戻る」をクリック

**重要（「外部」を選んだ場合）**: テスト中は最大100人までテストユーザーを追加できます。本番運用時は「公開ステータス」を「本番」に変更することで、全Googleユーザーがログイン可能になります。ただし、usersシートに登録されたメールアドレスのみがシステムを使用できるので安全です。

### 1-4. OAuth クライアントID を作成する

1. 左メニューから「APIとサービス」→「認証情報」をクリック
2. 上部の「+ 認証情報を作成」→「OAuthクライアントID」をクリック
3. アプリケーションの種類: 「ウェブ アプリケーション」を選択
4. 名前: `作文評価フロントエンド`
5. **「承認済みの JavaScript オリジン」に以下を追加**:
   - 「+ URI を追加」をクリック → `http://localhost:5173` と入力（テスト用）
   - 「+ URI を追加」をクリック → あとでNetlifyのURLを追加します
6. 「作成」をクリック
7. **表示された「クライアントID」をメモ帳にコピーしてください**
   （例: `123456789-xxxxxxxxx.apps.googleusercontent.com`）
   ※ これを後で `VITE_GOOGLE_CLIENT_ID` として使います

### 1-5. API を有効化する

1. 左メニューから「APIとサービス」→「有効なAPIとサービス」をクリック
2. 上部の「+ APIとサービスを有効にする」をクリック
3. 検索バーに `Google Sheets API` と入力 → 選択 → 「有効にする」をクリック
4. もう一度「+ APIとサービスを有効にする」をクリック
5. 検索バーに `Google Drive API` と入力 → 選択 → 「有効にする」をクリック

## ステップ2: スプレッドシートを作成する

これがシステムのデータベースになります。

### 2-1. 新規スプレッドシート作成

1. https://sheets.google.com/ を開く
2. 「空白」をクリックして新しいスプレッドシートを作成
3. スプレッドシートの名前を「作文評価データベース」に変更

### 2-2. スプレッドシートIDをメモする

ブラウザのアドレスバーのURLを確認してください:
```
https://docs.google.com/spreadsheets/d/ここがスプレッドシートID/edit
```
この「ここがスプレッドシートID」の部分をメモ帳にコピーしてください。
※ 後で `SPREADSHEET_ID` として使います

### 2-3. 権限を設定する

スプレッドシートの共有設定は**不要**です。
システムはデプロイ者（あなた）のアカウントでデータを読み書きするため、評価者にスプレッドシートの共有は必要ありません。

**注意**: 管理目的で直接データを確認・編集したい場合のみ、リーダーに「閲覧者」権限を付与してください。

## ステップ3: Google Drive フォルダを作成する

PDF（作文のスキャンデータ）を保存するフォルダです。

### 3-1. フォルダ作成

1. https://drive.google.com/ を開く
2. 「+ 新規」→「フォルダ」をクリック
3. フォルダ名: `作文PDFデータ`
4. 「作成」をクリック

### 3-2. フォルダIDをメモする

1. 作成したフォルダを開く
2. ブラウザのアドレスバーのURLを確認:
   ```
   https://drive.google.com/drive/folders/ここがフォルダID
   ```
3. 「ここがフォルダID」の部分をメモ帳にコピー
   ※ 後で `DRIVE_FOLDER_ID` として使います

### 3-3. フォルダの権限設定

PDF は Google Drive の iframe プレビューで表示されるため、評価者がPDFを閲覧できるよう共有設定が必要です。

**方法A: 全員に共有（推奨）**
1. フォルダ名を右クリック →「共有」→「共有」
2. 「一般的なアクセス」を「**リンクを知っている全員**」に変更
3. 権限は「**閲覧者**」を選択
4. 「完了」をクリック

**方法B: 特定ユーザーに共有**
1. フォルダ名を右クリック →「共有」→「共有」
2. 評価者全員のGoogleメールアドレスを入力
3. 権限は「**閲覧者**」でOK
4. 「送信」をクリック

※ Google Workspace 組織内の場合は「組織内の全員」に「閲覧者」がおすすめです

## ステップ4: Apps Script にコードを貼り付ける

### 4-1. Apps Script エディタを開く

1. ステップ2で作成したスプレッドシートを開く
2. メニューの「拡張機能」→「Apps Script」をクリック
3. Apps Script エディタが新しいタブで開きます

### 4-2. コードファイルを作成する

最初から「コード.gs」というファイルがあります。これを全て削除して、以下の手順で15個のファイルを作成します。

**ファイルの作成方法:**
1. 左側の「ファイル」パネルで「+」ボタン →「スクリプト」をクリック
2. ファイル名を入力（.gs は自動で付きます）
3. このPCの `gas/` フォルダにあるファイルの中身をコピー＆ペースト

**作成するファイル一覧**（この順番で作成してください）:

| # | ファイル名 | 説明 |
|---|-----------|------|
| 1 | `Config` | 設定ファイル |
| 2 | `SheetHelper` | データ操作の基本機能 |
| 3 | `Auth` | ログイン認証 |
| 4 | `Code` | メインの処理振り分け |
| 5 | `UserApi` | ユーザー管理 |
| 6 | `RoundApi` | 評価回管理 |
| 7 | `EssayApi` | 作文管理 |
| 8 | `AssignmentApi` | 割当管理 |
| 9 | `ScoreApi` | 評価（採点）管理 |
| 10 | `RubricApi` | ルーブリック管理 |
| 11 | `AvailabilityApi` | 担当可能数管理 |
| 12 | `NotificationApi` | 通知管理 |
| 13 | `TrainingApi` | デモ研修管理 |
| 14 | `ExportApi` | データ出力 |
| 15 | `AdminApi` | 管理者ツール |

※ 最初からある「コード.gs」は削除してかまいません（Code.gs で置き換え）

### 4-3. Config.gs を編集する

Config.gs の中に以下の設定箇所があります。メモした値に書き換えてください:

```javascript
var CONFIG = {
  SPREADSHEET_ID: 'ステップ2でメモしたスプレッドシートID',
  DRIVE_FOLDER_ID: 'ステップ3でメモしたフォルダID',
  GOOGLE_CLIENT_ID: 'ステップ1でメモしたクライアントID',
  // 以下はそのまま
  LOCK_WAIT_MS: 10000,
  LOCK_EXPIRE_MINUTES: 30,
  CACHE_TTL_SECONDS: 21600,
  PAGE_SIZE: 50,
};
```

### 4-4. 保存する

- `Ctrl+S`（Mac: `Cmd+S`）で保存
- 全ファイルにエラー表示がないことを確認

## ステップ5: Apps Script をデプロイする

### 5-1. シートの初期設定を実行する

1. Apps Script エディタ上部の関数選択ドロップダウンを開く
2. `handleAdminSetupSheets` を選択
3. ただし、これは直接実行できないので代わりに以下の手順:
   - Code.gs に一時的にテスト関数を追加:
   ```javascript
   function setupTest() {
     var adminUser = { id: 0, display_name: 'Admin', role: 'admin' };
     var result = handleAdminSetupSheets({}, adminUser);
     Logger.log(result);
   }
   ```
4. 関数選択で `setupTest` を選択 →「実行」ボタンをクリック
5. 初回実行時は権限の確認画面が出ます:
   - 「権限を確認」をクリック
   - 自分のGoogleアカウントを選択
   - 「詳細」→「（安全でないページ）に移動」をクリック
   - 「許可」をクリック
6. 実行ログに「XX個のシートを作成しました」と表示されればOK
7. スプレッドシートを開いて、シートのタブが増えていることを確認
8. テスト関数は削除してください

### 5-2. ウェブアプリとしてデプロイする

1. Apps Script エディタ →「デプロイ」→「新しいデプロイ」をクリック
2. 左上の歯車アイコン → 「ウェブアプリ」を選択
3. 以下を設定:
   - 説明: `v1 初回デプロイ`
   - **次のユーザーとして実行**: 「**自分**」を選択（※ 必ず「自分」を選んでください）
   - **アクセスできるユーザー**: 「**全員**」を選択
4. 「デプロイ」をクリック
5. **表示された「ウェブアプリ URL」をメモ帳にコピーしてください**
   （例: `https://script.google.com/macros/s/xxxxxxxxxxxxxxx/exec`）
   ※ これを後で `VITE_GAS_URL` として使います
6. 「完了」をクリック

### 5-3. 定期実行タイマーを設定する

期限切れのロックを自動で清掃する機能です。

1. Apps Script エディタ → 左メニューの時計アイコン（「トリガー」）をクリック
2. 右下の「+ トリガーを追加」をクリック
3. 以下を設定:
   - 実行する関数: `scheduledCleanExpiredLocks`
   - 実行するデプロイを選択: `Head`
   - イベントのソースを選択: `時間主導型`
   - 時間ベースのトリガーのタイプを選択: `分ベースのタイマー`
   - 分の間隔を選択: `10分ごと`
4. 「保存」をクリック

## ステップ6: フロントエンドを公開する（Netlify）

### 6-1. Node.js をインストールする（未インストールの場合）

1. https://nodejs.org/ を開く
2. 「LTS」バージョンをダウンロード
3. ダウンロードしたファイルを実行してインストール
4. インストール後、ターミナル（Mac）またはコマンドプロンプト（Windows）を開く
5. `node --version` と入力して Enter → バージョンが表示されればOK

### 6-2. 環境変数ファイルを作成する

1. このプロジェクトの `client` フォルダを開く
2. `client` フォルダ内に `.env` という名前のファイルを新規作成
3. 以下の内容を書き込む（値はメモした実際の値に置き換え）:

```
VITE_GAS_URL=https://script.google.com/macros/s/xxxxxxxxxxxxxxx/exec
VITE_GOOGLE_CLIENT_ID=123456789-xxxxxxxxx.apps.googleusercontent.com
```

### 6-3. ビルドする

ターミナルで以下のコマンドを1つずつ実行:

```bash
cd パス/WEBアプリ開発/client
npm install
npm run build
```

`dist` フォルダが作成されればビルド成功です。

### 6-4. Netlify にデプロイする（方法A: ドラッグ＆ドロップ）

**最も簡単な方法です:**

1. https://app.netlify.com/ を開く
2. Netlifyアカウントを作成（GitHubアカウントでサインアップ推奨）
3. ログイン後、画面下部に「Want to deploy a new site without connecting to Git? Drag and drop your site output folder here」というエリアがあります
4. `client/dist` フォルダを、このエリアにドラッグ＆ドロップ
5. 自動でデプロイが開始されます
6. 完了すると URL が発行されます（例: `https://xxxxxx.netlify.app`）

### 6-5. Netlify に環境変数を設定する

1. Netlify ダッシュボードでサイトをクリック
2. 「Site configuration」→「Environment variables」をクリック
3. 「Add a variable」をクリックして以下を追加:
   - Key: `VITE_GAS_URL` / Value: Apps ScriptのウェブアプリURL
   - Key: `VITE_GOOGLE_CLIENT_ID` / Value: OAuthクライアントID
4. **重要**: 環境変数を追加したら、再度ビルド＆デプロイが必要です
   - `npm run build` を再実行
   - `dist` フォルダをNetlifyに再度ドラッグ＆ドロップ

### 6-6. Google Cloud Console にNetlifyのURLを追加する

1. https://console.cloud.google.com/ を開く
2. 「APIとサービス」→「認証情報」→ 作成したOAuthクライアントIDをクリック
3. 「承認済みの JavaScript オリジン」に Netlify の URL を追加
   （例: `https://xxxxxx.netlify.app`）
4. 「保存」をクリック

## ステップ7: 初期ユーザーを登録する

### 7-1. users シートに直接記入する

1. スプレッドシートを開く
2. `users` シートのタブをクリック
3. ヘッダー行（1行目）の下に以下の情報を記入:

| id | login_id | display_name | email | role | is_active | created_at |
|----|----------|-------------|-------|------|-----------|------------|
| 1 | leader01 | リーダー太郎 | leader@example.com | leader | TRUE | 2024-01-01T00:00:00.000Z |
| 2 | evaluator01 | 評価者花子 | evaluator@example.com | evaluator | TRUE | 2024-01-01T00:00:00.000Z |

**重要**:
- `email` 列には、各ユーザーの**Googleアカウントのメールアドレス**を正確に入力してください
  （ユーザーはこのメールアドレスのGoogleアカウントでログインします）
- `role` は `leader`（リーダー）または `evaluator`（評価者）のどちらか
- `id` は連番（1, 2, 3, ...）

**ログイン後の追加登録**: 最初のリーダーアカウントでログインすると、「ユーザー管理」メニューから追加のユーザーを画面上で登録できます。CSV一括登録にも対応しています。

## よくある問題と解決方法

### 「ログインできない」
- Google Cloud Console の OAuth 同意画面で「外部」を選んだ場合、テストユーザーとしてメールアドレスを追加する必要があります
- 「承認済みの JavaScript オリジン」に正しいURL（`https://xxxxxx.netlify.app`）が設定されているか確認
- `users` シートに正しいメールアドレスが登録されているか確認

### 「403エラーが出る」
- `users` シートに該当ユーザーのメールアドレスが登録されているか確認
- `is_active` が `TRUE` になっているか確認

### 「PDFが表示されない」
- Google Drive フォルダの共有設定で、評価者に「閲覧者」以上の権限があるか確認
- `essays` シートの `pdf_file_id` 列に正しいファイルIDが入っているか確認

### 「データがおかしい・エラーが出る」
- Apps Script エディタ → 左メニュー「実行」→ エラーログを確認
- スプレッドシートの各シートのヘッダー行（1行目）が正しいか確認

### 「GASの更新を反映したい」
1. Apps Script エディタでコードを修正・保存
2. 「デプロイ」→「デプロイを管理」
3. 該当デプロイの鉛筆マーク → バージョンを「新しいバージョン」に変更
4. 「デプロイ」をクリック
※ URLは変わりません

### 「Netlifyの更新を反映したい」
1. `client` フォルダで `npm run build` を実行
2. `client/dist` フォルダをNetlifyにドラッグ＆ドロップで再デプロイ
